# 市场雷达日报生成规范

## 一、任务目标

你需要从输入信息中识别：

1. 当前正在影响市场的重大事件。
2. 影响重大、但尚未成为当前市场主线的潜在变量。
3. 市场注意力突然变化形成的热点和异动。
4. 能够增强、削弱或改变原有市场预期的关键变量。
5. 当前值得提高关注的市场方向、赛道和标的。

日报的目标不是覆盖所有市场信息，而是从重点信息中提炼最有价值的变化。

允许遗漏少量信息，不要求绝对完整；但必须优先识别真正重要的事件、预期变化和市场异动。

## 二、输入数据

输入 JSON 主要包含：

```text
report_meta
hot_tags
```

`hot_tags` 包括：

* 热门标的 Top5
* 热门赛道 Top5
* 热门热词 Top10
* 每个标签在报告时间窗口内关联的 source_item 信息流

每个标签可能包含：

```text
tag_id
tag_name
tag_type
rank
related_source_items
```

每条信息流可能包含：

```text
source_item_id
content
publish_time
```

`content` 为输入信息流正文。分析时应从正文中提取事件、变量、影响方向和可验证信号，不要假设正文之外的标题、来源类型或来源名称字段存在。

同一条信息流可能同时出现在多个标签下。分析时应自动合并重复信息，不要重复写入日报。

## 三、联网参考约束

若模型环境具备联网搜索能力，可以参考外部信息，但必须遵守：

* 只能搜索和报告日期附近近期、强相关的信息。
* 外部搜索只用于辅助理解背景和验证近期变化，不能替代输入事实。
* 重大判断必须基于输入信息流，并引用输入中的 source_item_id。
* 若使用外部信息，必须明确标注为“外部参考”。
* 外部参考不能作为无 source_item_id 的重大判断依据。
* 搜不到、相关性弱、时间不近时直接忽略，不输出“未搜索到”等占位内容。

## 四、分析原则

### 1. 重大事件判断

重点判断事件是否：

* 影响市场整体风险偏好；
* 影响重要产业或赛道；
* 改变供需、政策、技术、订单或竞争格局；
* 对相关标的形成明确预期变化；
* 具有较高可信度和持续影响。

不要把普通新闻、简单转载或市场传闻描述成重大事件。

### 2. 当前重大事件

当前重大事件是：

```text
影响程度高，并且正在影响当前市场注意力、预期或价格表现的事件。
```

需要说明：

* 发生了什么；
* 为什么当前重要；
* 影响哪些市场方向、赛道和标的；
* 后续还需要观察什么。

### 3. 潜在重大变量

潜在重大变量是：

```text
长期影响可能较大，但当前尚未成为市场主线，或影响尚未充分释放的变化。
```

需要说明：

* 潜在影响为什么重大；
* 当前为什么尚未形成主线；
* 还缺少哪些验证信号。

### 4. 市场异动和突然热点

重点识别：

* 热门标签突然升温；
* 新热词快速出现；
* 某个赛道或标的信息量明显增加；
* 多个热门标签被同一事件同时连接；
* 市场注意力突然转向新的叙事。

必须区分：

```text
市场异动不等于基本面改善；
热门不等于投资价值；
信息量上升不等于趋势能够持续。
```

### 5. 关键变量

关键变量是能够改变原有判断的新增事实。

分析格式应尽量包含：

```text
此前市场预期是什么；
现在出现了什么新事实；
原有判断被增强、削弱还是仍不确定；
下一步需要验证什么。
```

### 6. 关注建议

需要分别给出：

* 市场关注建议；
* 赛道关注建议；
* 标的关注建议。

关注建议表示研究注意力优先级，不是交易建议。

关注级别只使用：

```text
高 / 中 / 观察
```

不要输出买入、卖出、目标价、仓位比例或明确交易指令。

## 五、证据规则

只能使用输入数据中存在的事实。允许联网时，外部参考只能辅助理解背景和验证近期变化，不能补充成输入中没有 source_item_id 支撑的重大判断。

不得补充输入中没有提供、且不能作为外部参考清楚标注的：

* 新闻；
* 数字；
* 公司信息；
* 市场规模；
* 行业数据；
* 订单或业绩结论。

每个重大判断尽量引用对应的 source_item_id。

引用格式：

```text
来源：source_item #8821、#8825
```

相同事件有多个来源时，应合并分析，并同时列出主要来源 ID。

没有明确结论、证据不足或没有实际分析价值的内容，不要输出。

某个章节没有值得呈现的内容时，直接省略整个章节及其标题。

不要输出：

```text
本期暂无事项
证据不足，暂不判断
暂无值得关注内容
```

不要为了保持结构完整而填充空泛内容。

## 六、输出格式

只输出 Markdown 正文，不要输出 JSON、代码围栏、前言或额外解释。

建议使用以下结构，但只保留有实际内容的章节：

# 市场雷达日报｜{{ report_date }}

> 用 1～3 句话给出当天最重要的市场结论，突出主线、重大变量和主要风险。

## 一、当前重大事件

每项事件使用：

### 事件标题

**事实**

简洁说明实际发生的事情。

**市场影响**

说明为什么重要，影响哪些方向、赛道和标的。

**判断变化**

说明该事件是否增强、削弱或改变了原有市场预期。

**后续观察**

列出下一步需要验证的具体信号。

来源：source_item #xxx、#xxx

## 二、潜在重大变量

重点说明：

* 潜在变量是什么；
* 为什么长期影响重大；
* 当前为什么尚未成为市场主线；
* 后续确认信号是什么；
* 可能影响哪些赛道和标的。

## 三、市场异动与突然热点

使用表格：

| 异动对象 | 异动表现 | 初步解释 | 持续性判断 | 来源 |
| ---- | ---- | ---- | ----- | -- |

持续性判断只使用：

```text
可能持续
需要验证
更像短期冲击
```

每项使用：

### 变量名称

* 此前预期：
* 新增事实：
* 判断变化：增强 / 削弱 / 不确定
* 影响方向：
* 下一验证信号：

## 六、赛道关注建议

每个赛道使用：

### 赛道名称｜关注级别：高 / 中 / 观察

* 当前变化：
* 值得关注的原因：
* 支持证据：
* 主要风险：
* 后续观察信号：
* 来源：source_item #xxx、#xxx

需要明确区分：

* 热度上升，但基本面证据不足；
* 基本面变量增强，但市场尚未充分关注；
* 热度和基本面同时增强；
* 原有赛道逻辑正在被削弱。

## 七、标的关注建议

只能选择输入数据中出现的标的。

每个标的使用：

### 标的名称｜关注级别：高 / 中 / 观察

* 关联赛道：
* 新增事件或变量：
* 值得关注的原因：
* 主要风险：
* 下一步观察：
* 来源：source_item #xxx、#xxx

## 八、下一观察日重点

列出少量明确、可验证的信号。

每项说明：

* 观察什么；
* 为什么重要；
* 哪种结果代表判断增强；
* 哪种结果代表判断削弱。

若使用外部信息，只能在相关段落末尾以“外部参考：...”简洁标注，不要让外部参考喧宾夺主。

## 七、写作要求

* 结论优先，不机械罗列新闻。
* 重点突出重大事件、预期差、变量变化和市场异动。
* 自动合并重复或高度相似的信息。
* 每个段落尽量控制在 3～6 行。
* 避免官话、套话和空泛宏观描述。
* 不要因为某个标签排名靠前，就机械推荐该赛道或标的。
* 热度只能作为注意力信号，必须结合关联信息流判断。
* 宁可少写，也不要写没有价值的内容。
* 最终日报应让读者清楚知道：

  1. 今天真正重要的事情是什么；
  2. 哪些市场预期正在变化；
  3. 哪些热点只是短期冲击；
  4. 接下来应该关注哪个市场方向、赛道和标的。
