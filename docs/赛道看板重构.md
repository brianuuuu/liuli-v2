请根据我提供的“赛道看板.png”参考图，重新设计赛道发现的赛道看板页面。

页面定位：
赛道看板是“赛道发现”一级模块下的首页，用于从市场热度与研究判断中识别值得重点跟踪的赛道。
它不是标签管理页，也不是赛道详情页，而是每日打开后先看的赛道驾驶舱。

页面整体风格：
- 克制、专业、数据看板感

赛道看板-页面主要区块：

1. 顶部标题区
标题：赛道看板

2. 顶部统计卡片
展示 4 个核心指标：
- 升温赛道数量
- 重点跟踪赛道数量
- 待确认动态数量
- 今日最高热度赛道

卡片需要包含图标、主数值、简短说明。

3. 赛道热度趋势图
使用 ECharts 折线图。
展示多个赛道最近 7，30，90 天热度趋势

4. 今日赛道热度榜
字段：
- 排名
- 赛道
- 当前热度
- 7、30、90日变化
- 阶段
- 赛道评分

阶段可选值：
- 概念期
- 验证期
- 成长期
- 过热期
- 衰退期

5. 重点赛道卡片

每张卡片包含：
- 赛道名称
- 赛道评分
- 当前判断 current_view
- 阶段 stage
- 置信度 confidence_level
- 绑定标的数量
- 最近动态数量

6. 最新赛道动态
展示来自 track_material 的材料流。
字段：
- 时间
- 赛道
- 材料类型：信息流 / 笔记
- 方向：支持 / 削弱 / 中性 / 噪音
- 摘要
- 重要性


7. AI 赛道分析摘要
底部展示当前选中赛道的 AI 分析摘要。
内容包括：
- 未来市场空间
- 当前市场规模
- 当前增长速度
- 当前热度解释
- 主要机会
- 主要风险
- 观察信号

这些数据来自 track_analysis_snapshot。

数据模型约束：
- 赛道基础信息来自 track
- 赛道标签来自 track_tag_relation
- 赛道绑定标的来自 stock_track_relation
- 赛道动态来自 track_material
- 赛道 AI 分析来自 track_analysis_snapshot
- 赛道热度由 track_tag_relation + tag_heat_snapshot 聚合得到
