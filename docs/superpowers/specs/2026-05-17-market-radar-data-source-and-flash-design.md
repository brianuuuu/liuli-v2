# Market Radar Data Source And Flash Design

## Decision

市场雷达下拆成两个二级模块：

- `数据源`：底层 `source_item` 管理视图，展示全量原始信息源，保留手动新增和详情。
- `快讯`：新闻阅读视图，使用时间轴模式展示 `source_item` 中的新闻、政策、舆情和重大公告。

## Rationale

`source_item` 是市场雷达的底层信息源表，不等同于用户日常阅读的新闻快讯。数据源视图偏管理、排错、校验；快讯视图偏阅读、筛选、发现信号。两个入口并列可以让高频阅读页面保持干净，同时不丢失底层数据管理能力。

## UI Placement

```text
市场雷达
├── 总览
├── 热度榜
├── 数据源
├── 快讯
├── 标签
├── 候选标签
└── 关系图
```

## Flash View

快讯页参考老平台新闻中心：

- 左侧主内容是时间轴。
- 顶部操作区提供 `全部 / 重要` 切换和 `同步财联社`。
- 右侧提供关键词、来源、类型筛选。
- 每条快讯展示发布时间、来源、类型、标题、正文摘要。
- 标签展示先按当前标签库做前端文本命中，后续可切换为后端 `source_tag` 关系。

## Data Model

不新增表，不改 `source_item` 表结构。财联社同步仍写入：

- `source_type = news`
- `source_name = 财联社`
- `title`
- `content`
- `publish_time`

## Scope

本阶段只做 Web 入口和时间轴展示拆分，不新增独立新闻中心，不新增配置项。
