# Android 卡片阴影与 Markdown 表格修复设计

## 目标

- 移动端 H5 的普通内容卡片取消阴影，以边框区分卡片和页面背景。
- Markdown 报告详情正确渲染 GitHub Flavored Markdown 表格。
- 宽表格只在表格区域横向滚动，不造成整个页面横向溢出。

## 范围

本次仅修改 `invest_assistant/ui/android/h5`。不修改 Web、Android 原生业务层、后端接口或数据库。

## 样式设计

全局普通内容卡片通过现有 `--shadow` 变量统一取消阴影。依赖该变量的指标卡片、区块卡片、笔记卡片、预警卡片和推荐词卡片均不再显示投影，继续保留现有边框、圆角和白色面板背景。

浮动新增按钮、操作反馈等需要表达悬浮层级的控件使用独立阴影声明，不受本次调整影响。桌面调试视口外框也不属于业务内容卡片，不纳入调整。

## Markdown 渲染设计

Android H5 引入与 Web 端一致的 `remark-gfm`，在报告详情的 `ReactMarkdown` 上注册该插件。标准 GFM 表格语法将生成 `table`、`thead`、`tbody`、`th` 和 `td` 元素，而不是普通段落文本。

移动端表格保留边框和紧凑单元格样式。表格宽度超过内容区域时，横向滚动限制在表格自身；页面主体继续禁止横向溢出。单元格默认不换行，保证列数据的可读性。

## 测试与验收

- 组件测试使用包含表头和数据行的 Markdown，验证生成表格及对应单元格。
- 样式检查验证全局普通内容卡片阴影变量为 `none`，并确认浮动操作控件仍保留独立阴影。
- 运行 Android H5 Vitest、TypeScript 类型检查和生产构建。
- 不运行数据库测试或任何修改数据库数据的命令。
