# Console Job Center Card Mode Design

## Goal

将控制台任务中心从表格主列表改为紧凑卡片工作台。页面仍服务运维操作，优先保证状态扫描、手动运行、配置、日志查看路径短。

## Scope

- 仅调整 Web 前端任务中心页面。
- 不新增后端接口，不改任务表结构。
- 继续使用现有 `listJobs`、`listRunRequests`、`listJobLogs`、`syncJobDefinitions`、`runJob`、`updateJob`。
- 保留现有任务配置弹窗、手动运行弹窗、任务详情抽屉、运行请求和执行日志表格。

## Layout

任务中心采用“密集任务卡 + 运行详情区”：

- 顶部工具栏保留 `同步任务定义`。
- 工具栏增加前端筛选：关键词、模块、状态、启用状态。
- 主列表使用响应式任务卡网格，桌面 2-3 列，窄屏 1 列。
- 卡片显示任务名、任务编码、模块、触发类型、启用状态、最近状态、最近运行时间、超时、重试次数。
- 卡片操作：`运行`、`配置`、`日志`、`详情`。
- 点击卡片或 `日志` 选中任务并加载该任务日志。
- 下方保留 `运行请求 / 执行日志` tabs。选中任务时只看该任务；未选中时运行请求展示全部，执行日志为空。

## Visual Rules

- 遵循 `docs/superpowers/specs/2026-05-16-liuli-web-ui-spec.md`。
- 卡片是紧凑运维列表项，不使用营销卡片风格。
- 使用 7px 左右低圆角、细边框、浅色背景分层。
- 失败状态只用轻量红色状态点和边框提示，不整卡大面积染色。
- Dark mode 使用现有 CSS 变量继承，首轮只做基础兼容。

## Behavior

- 筛选全部在前端完成。
- 模块选项从任务数据中去重生成。
- 状态选项包含 `success`、`running`、`failed`、`error`、`completed` 和 `未运行`。
- 启用状态支持 `全部`、`启用`、`停用`。
- 筛选结果为空时显示当前通用空状态。
- 任务运行、配置、同步定义后的刷新逻辑沿用现有 `refreshAll()`。

## Acceptance

- 任务中心主列表不再是表格。
- 任务卡能直接运行、配置、查看日志和打开详情。
- 选中任务后运行记录标题随任务变化。
- 筛选不会请求新接口。
- `npm.cmd run build` 通过。
