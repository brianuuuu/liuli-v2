# Console Job Run Event List Design

## Goal

将控制台任务中心右侧“运行记录”从嵌套表格改成事件流列表，减少卡片内表格的拥挤感，同时保留运维需要的关键字段。

## Scope

- 仅调整 Web 前端任务中心右侧运行记录区域。
- 不新增后端接口，不修改任务中心表结构。
- 保留 `运行请求 / 执行日志` 两个 Tabs。
- 保留当前任务卡、运行、配置、详情和日志加载逻辑。

## Event List Shape

### 运行请求

每条运行请求展示为轻量事件行：

- 状态点和状态 Tag。
- `请求 #id`。
- 任务名。
- 请求时间。
- 错误摘要，仅在存在错误时展示。
- 点击事件行展开，展示 `params_json`、开始时间、结束时间、请求人。

### 执行日志

每条执行日志展示为轻量事件行：

- 状态点和状态 Tag。
- 开始时间和耗时。
- 一行指标：抓取、处理、新增、更新。
- 错误摘要，仅在存在错误时展示。
- 点击事件行展开，展示 `params_json`、`result_json`、完整错误信息。

## Visual Rules

- 不使用嵌套表格。
- 事件行使用细边框、低圆角、浅背景，符合当前紧凑工具台风格。
- 状态点用颜色表达结果：成功绿色、运行蓝色、失败红色、未知灰色。
- 详情内容使用轻量 key/value 栅格或 pre-wrap 文本，不引入新视觉体系。
- 空状态继续使用现有 `EmptyAction`。

## Acceptance

- 任务中心右侧 Tabs 内不再出现 Table。
- 运行请求和执行日志都以事件流列表展示。
- 点击事件行可以展开或收起详情。
- `npm.cmd run build` 通过。
- 浏览器验证右侧运行记录视觉不再是卡片嵌套表格。
