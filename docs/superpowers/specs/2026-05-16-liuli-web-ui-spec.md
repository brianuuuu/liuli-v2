# 琉璃 Web UI 规范

## 1. 适用范围

本文档定义琉璃 Web 端后续开发必须遵守的 UI 规范，适用于 `invest_assistant/ui/web` 下的所有页面、组件和业务模块。

当前 Web UI 采用 **方案 C：紧凑终端工具台**。核心目标是：信息密度高、线条清晰、区域克制、操作路径短，避免传统后台的厚重卡片、强胶囊按钮和大面积说明文案。

后续开发应优先复用现有组件：

- `components/layout/AppLayout.tsx`
- `components/layout/TopStatusBar.tsx`
- `components/layout/ModuleTabs.tsx`
- `components/common/PageHeader.tsx`
- `components/common/WorkbenchCard.tsx`
- `components/common/RecordTable.tsx`
- `components/common/EmptyAction.tsx`
- `components/charts/ChartCard.tsx`

不要在单个页面里重新发明布局、线条、按钮、表格或卡片样式。

## 2. 设计原则

### 2.1 工作台，不是营销页

- 第一屏必须是可操作的研究工作台。
- 不使用 Hero、宣传文案、大插画、装饰性背景图。
- 页面说明文字要压缩成短标签上下文。

示例：

```text
市场雷达  信号 / 热度 / 关系
标的分析  标的池 / 评分 / 报告
控制台    任务 / 数据 / 配置
```

不要写成长句：

```text
市场雷达
发现市场正在关注什么
```

### 2.2 细线分区

- 页面区域通过 `1px` 细线、浅色背景差异和小间距区分。
- 不使用厚边框、重阴影、强渐变。
- 外层应用壳有闭合边界，内部通过细线继续分区。

### 2.3 控件分级

不是所有按钮都应该有胶囊背景。

- 输入、筛选、状态：可以有轻边框块。
- 主题、刷新、用户、更多：默认无背景工具项。
- 主要动作：使用 Ant Design `primary`，但只在需要明确提交时使用。
- 危险动作：使用 `danger`，并配合确认。

### 2.4 模块边界

Web 一级导航必须保持：

- 总览
- 市场雷达
- 赛道发现
- 标的分析
- 预警中心
- 组合管理
- 知识库
- 控制台

控制台是操作面板，不拥有业务能力。比如标签库页面可以在控制台，但标签能力归 `market_radar`。

## 3. 字体规范

### 3.1 字体族

全局字体必须保持：

```css
font-family: Inter, "Segoe UI", "Microsoft YaHei", system-ui, sans-serif;
```

不要在局部页面引入新的字体族。

### 3.2 字号

| 用途 | 字号 | 字重 | 说明 |
|---|---:|---:|---|
| 一级页面标题 | 23px | 700/800 | `PageHeader` 标题 |
| 顶部工具条文字 | 12px | 500 | 状态、工具项 |
| 页面上下文标签 | 13px | 400 | 标题右侧短标签 |
| 卡片标题 | AntD 默认，约 14px | 700 | `WorkbenchCard` |
| 表格表头 | AntD small，约 14px | 600 | 不加大 |
| 指标数值 | 27px | 800 | 指标面板核心数字 |
| 指标说明 | 12px | 400 | 指标补充信息 |

禁止在卡片、表格、侧栏中使用 Hero 级大标题。

## 4. 色彩规范

### 4.1 浅色模式 Token

当前浅色模式以冷灰蓝工作台为基调。

```css
--ll-border: #e2e8f0;
--ll-border-soft: #f1f5f9;
--ll-panel: #ffffff;
--ll-panel-subtle: #f8fafc;
--ll-surface: #f4f7fa;
--ll-text: #0f172a;
--ll-muted: #64748b;
--ll-accent: #2563eb;
--ll-accent-soft: #eff6ff;
```

使用规则：

- 页面外背景：`#eaeff5` 到 `#e2e8f0` 的轻微垂直层次。
- 应用壳：`--ll-panel`。
- 内容区：`--ll-surface`。
- 卡片/表格主体：`--ll-panel`。
- 卡片头/表头/轻块：`--ll-panel-subtle`。
- 主文字：`--ll-text`。
- 次要文字：`--ll-muted`。
- 选中态与主操作：`--ll-accent`。

不要新增大面积紫色、米色、深蓝、棕橙、强渐变或高饱和背景。

### 4.2 深色模式 Token

深色模式第一版已支持功能切换，后续开发必须保持可用，不得写死浅色颜色。

```css
[data-theme="dark"] {
  --ll-border: rgba(255, 255, 255, 0.08);
  --ll-border-soft: rgba(255, 255, 255, 0.03);
  --ll-panel: #161b22;
  --ll-panel-subtle: #21262d;
  --ll-surface: #0d1117;
  --ll-text: #e6edf3;
  --ll-muted: #8b949e;
  --ll-accent: #58a6ff;
  --ll-accent-soft: rgba(56, 139, 253, 0.15);
}
```

深色约束：

- 新组件必须使用 CSS 变量或 AntD token。
- 不允许写死 `#ffffff` 作为文字颜色之外的背景，除非有对应深色覆盖。
- 图表必须通过 `ChartCard` 或主题辅助函数处理背景、文字和网格线。
- 深色模式不作为当前视觉精调重点，但不能出现不可读文本、白底黑字突兀块、透明层叠错误。

### 4.3 状态色

| 状态 | 颜色 | 使用场景 |
|---|---|---|
| 成功/在线 | `#10b981` | API 在线、小圆点 |
| 警告/待同步 | `#f97316` | 任务待同步 |
| 主操作 | `--ll-accent` | 选中态、主按钮 |
| 危险 | AntD danger | 删除、停用、拒绝 |
| 辅助 | `--ll-muted` | 描述、提示、弱信息 |

状态色只用于点状提示、文字、少量强调，不用于大面积背景。

## 5. 布局规范

### 5.1 应用外壳

根页面保留 7px 外边距和闭合应用壳。

```css
body {
  min-width: 960px;
  padding: 7px;
}

.app-shell {
  border-radius: 9px;
  overflow: hidden;
}
```

页面整体应像一个完整工作台窗口。不要在模块内容区再创建第二层大外框。

### 5.2 左侧一级导航

侧栏宽度：

- 展开：208px
- 收起：66px

规则：

- 侧栏支持展开/收起，状态保存到 `localStorage`。
- 展开时显示 Logo、`琉璃 Liuli`、副标题和菜单文字。
- 收起时只显示 Logo 和图标。
- 底部放收起/展开按钮。
- 一级导航只放主模块，不放二级菜单。

侧栏菜单：

- 高度：38px
- 圆角：6px
- 普通态无边框、无强背景
- Hover 使用 `--ll-panel-subtle`
- 选中使用 `--ll-accent-soft` 和 `--ll-accent`

### 5.3 顶部工具条

顶部工具条高度固定为 38px。

工具条内容分级：

- 左侧：状态和搜索
  - `API 在线`
  - `任务 2 待同步`
  - 全局搜索
- 右侧：轻量工具项
  - 浅色/深色切换
  - 刷新
  - 用户
  - 更多

约束：

- 顶部栏只用底部分割线，不单独做胶囊外框。
- 搜索框和状态块可以有轻边框。
- 右侧普通工具项必须无背景，hover 时才出现浅背景。
- 工具项之间可使用 1px 竖线分隔。

### 5.4 内容区

内容区使用：

```css
.app-content {
  padding: 14px 20px 24px;
  background: var(--ll-surface);
}
```

不要把内容区背景改成纯白。纯白应留给卡片和表格主体。

### 5.5 一级模块标题区

一级模块标题使用 `PageHeader`。

规则：

- 标题保留。
- 简介必须改成短标签上下文。
- 标题和上下文同一行。
- 不写长说明文案。

示例：

```tsx
<PageHeader title="市场雷达" description="信号 / 热度 / 关系" />
```

## 6. 导航与 Tabs

### 6.1 一级导航

一级导航只在左侧栏。

不要在内容区重复一级导航。

### 6.2 二级 Tabs

一级模块内部功能通过内容区顶部横向 Tabs 切换。

Tabs 规则：

- 底部一条 `1px` 分隔线。
- Tab 间距约 26px。
- 选中态使用蓝字 + 2px 细下划线。
- 不使用胶囊式 Tab。
- 不用卡片包裹 Tabs。

示例模块：

```text
市场雷达：总览 / 热度榜 / 信息源 / 标签 / 候选标签 / 关系图
控制台：系统状态 / 任务中心 / 标签库 / 候选标签 / 报告库 / 公告财报库 / 股票基础库 / 系统配置 / AI 日志
```

## 7. 卡片与面板

### 7.1 WorkbenchCard

业务页面的主要容器使用 `WorkbenchCard`。

规则：

- 圆角：7px
- 边框：`1px solid var(--ll-border)`
- 背景：`var(--ll-panel)`
- 默认无阴影
- 卡片头背景：`var(--ll-panel-subtle)`
- 卡片头高度：约 40px

不要用嵌套卡片。重复列表项可以用卡片，但页面大区块不要卡片套卡片。

### 7.2 指标面板

指标面板用于模块总览。

规则：

- 四列栅格优先。
- 间距：10px。
- 面板背景：`var(--ll-panel)`。
- 数值字号：27px。
- 指标说明使用 12px。

指标不要用过强颜色。涨跌、状态才使用状态色。

## 8. 表格规范

表格使用 Ant Design `Table`，首选 `size="small"`。

规则：

- 表格外层有 1px 边框和 7px 圆角。
- 表头背景：`var(--ll-panel-subtle)`。
- 表头文字：`var(--ll-muted)`，字重 600。
- 行 hover：浅色模式用 `#f7fbff`。
- 空数据必须显示真实空状态，不允许假数据填充。
- 行点击查看详情时，优先使用右侧抽屉。

操作列规则：

- 常规操作：小按钮。
- 危险操作：`danger` + `Popconfirm`。
- 已处理或不可操作状态必须禁用按钮。

## 9. 按钮与操作

### 9.1 普通按钮

- 页面中的“新增、同步、运行”等明确命令使用按钮。
- 表格行内按钮使用 `size="small"`。
- 避免一行堆太多主按钮。

### 9.2 顶部工具项

顶部右侧工具项不是普通按钮视觉：

- `type="text"`
- 无边框
- 无默认背景
- hover 才出现轻背景

### 9.3 危险动作

停用、删除、拒绝等危险动作必须：

- 使用 `danger`
- 使用 `Popconfirm` 或二次确认
- 后端成功后刷新列表

## 10. 表单与弹窗

新增/编辑使用 `Modal` 或抽屉表单。

规则：

- 简单新增/编辑：`Modal`
- 内容较长、需上下文对照：右侧抽屉
- 表单使用 `layout="vertical"`
- 必填项必须有规则提示
- 提交成功后关闭弹窗并刷新列表

标签库示例规则：

- `stock` 标的标签由股票主数据同步，不允许手动新增、编辑、停用。
- 人工维护只允许 `track` 赛道标签和 `hotword` 热点词标签。

## 11. 空状态

空状态必须真实表达当前无数据，并提供下一步方向。

示例：

```text
暂无热度数据，去控制台同步或运行市场雷达任务
暂无公告财报
暂无候选标签
```

禁止使用假数据撑页面。

## 12. 图表规范

图表使用 ECharts。

规则：

- 通用图表容器使用 `ChartCard`。
- 图表背景跟随主题。
- 浅色模式下文字使用冷灰，网格线使用浅灰蓝。
- 深色模式下文字和网格线必须可读。
- K 线、分时、成交量图暂不实现；后续需要时再引入 `lightweight-charts`。

图表首版原则：

- 先保证信息准确和可读。
- 不做夸张动效。
- 不使用装饰性渐变背景。

## 13. 详情形态

规则：

- 列表浏览类：右侧抽屉详情。
- 复杂研究对象：独立详情页。
- 新增/编辑/确认：弹窗或抽屉表单。
- 报告：抽屉预览 + 独立阅读页。

不要在列表页跳转过多，除非对象本身需要长期研究空间，例如赛道、标的、组合。

## 14. 主题模式约束

### 14.1 模式

Web 必须支持：

- 浅色
- 深色
- 跟随系统

当前顶部工具条展示浅色/深色切换，底层主题能力仍由 `LiuliThemeProvider` 管理。

### 14.2 实现规则

- 主题状态保存到 `localStorage`。
- `document.documentElement` 和应用根节点都应设置 `data-theme`。
- 新样式必须使用 CSS 变量或 AntD token。
- 禁止只写浅色样式而不考虑深色。

### 14.3 浅色验收

浅色是当前主要验收主题。

浅色必须满足：

- 页面外层有完整工作台边界。
- 内容区为冷灰蓝，不是纯白。
- 卡片、表格、面板使用白色和极浅灰蓝分层。
- 线条细，不厚重。
- 页面无大面积强色块。

### 14.4 深色验收

深色第一阶段要求：

- 文本可读。
- 输入框、表格、卡片不出现突兀白底。
- 图表不空白、不反色错误。
- 操作按钮可识别。

深色细节可后续精修，但不能破坏功能。

## 15. 开发检查清单

新增或修改 Web 页面前，必须检查：

- 是否复用了 `AppLayout`、`PageHeader`、`ModuleTabs`、`WorkbenchCard`。
- 一级模块标题是否使用短标签上下文。
- 是否避免了胶囊化所有按钮。
- 是否避免了卡片套卡片。
- 表格是否使用 small 尺寸和真实空状态。
- 危险操作是否有确认。
- 是否使用 CSS 变量而不是硬编码浅色。
- 深色模式下是否可读。
- 是否没有引入 `lightweight-charts`。
- 是否通过 `npm.cmd run build`。

## 16. 不允许的做法

- 新增营销页式 Hero。
- 使用大面积渐变、装饰光斑、装饰图。
- 把所有按钮做成胶囊。
- 把二级 Tabs 做成按钮组。
- 在内容区重复一级导航。
- 为了页面好看填充假数据。
- 在页面里写死大量颜色而不支持深色。
- 把 `console` 当成业务能力归属地。
- 手动维护 `stock` 标的标签。
